form 태그 / input 태그

form 태그

        - 입력 양식을 작성하는 판(영역)
        - 내부에 작성된 입력값을 지정된 곳으로 제출하는 역할 수행
        
        [속성]
        1. action : 제출될 곳을 지정하는 속성(주소를 작성)
        2. method : 설정된 곳으로 데이터를 전달하는 방법을 정함
        3. name : 이름 지정하는 속성(JS 사용)
        4. target : 제출된 페이지를 현재 or 새 탭으로 열기

input 태그

        - 웹 문서에서 사용자(클라이언트)가 입력할 수 있는 양식을 제공하는 태그
        - 많은 type이 존재하고 input의 변형 형태도 다수 존재

        * input 태그에는 name 속성이 거의 필수로 작성되어야 함

Label 태그

        - 입력 관련 태그(input, select, textarea 등)의 제목을 나타내는 태그

        - 사용법 1 : label 태그 내부에 입력 관련 태그를 하나 작성하면 label 태그를 클릭해도
        입력관련 태그가 선택된다.

        - 사용법 2 : 입력 관련 태그의 id 속성 값을 label 태그의 for 속성 값에 똑같이 적으면 연결된다.


select / option 태그

        - select : 클릭하면 밑으로 내려오는 입력 태그(드롭 다운)
        - option : select 클릭 시 보여질 메뉴를 작성하는 태그를

        * select와 option은 input의 name, value가 분리된 태그라고 생각하면 된다.

textarea

        - 여러 줄을 입력할 수 있는 입력 상자

        [속성]
        - name : 제출된 곳에서 데이터를 읽을 때 필요한 이름
        - rows : textarea의 행 크기를 지정("rows=5" ->5행)
        - cols : textarea의 열 크기를 지정("cols=5" ->5열)
        - style="reasize:none" : 크기 변경 불가

        * textarea는 pre 태그처럼 작성된 모양 그대로 출력하는 성질을 지님.ㅇ

기타 입력 관련 태그

    - type="color" : 색을 출력할 수 있는 입력 상자를 출력
    - type="file" : 파일을 선택할 수 있는 탐색기 실행
    - type="hidden" : 화면에 보이지 않음(화면에 숨기고 싶은 값이 있을 경우에만 사용>)



fieldset / legend

        field set : 테두리를 만들어서 영역을 설정. 주로 form 태그 내부에서 입력 요소의 그룹을 지을때 사용
        legend(제목) : fieldset 테두리에 이름을 저장하는 태그

button 태그

        - 버튼을 만드는 태그
        - type 속성 값에 따라서 동작이 달라짐

        type="submit"
          

        type="reset" : 버튼이 포함된 form 태그의 모든 입력 값을 초기화

        type="button" : 아무런 기능이 없는 버튼. JS을 통해서 기능 추가.
